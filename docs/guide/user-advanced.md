# 利用者ガイド（発展機能）

このガイドは、自分でショップを持って運営したいプレイヤー、共同オーナー、あるいは通常の購入・納品以外のコマンドを使いこなしたいプレイヤー向けです。ショップの見つけ方・買い方・納品の基本は [利用者ガイド（基本機能）](user-basic.md) を参照してください。

## 1. 自分のショップを作る

### 1.1 スポーンエッグを入手する

プレイヤーショップは、管理者から配布される **専用スポーンエッグ** から作ります。エッグには「出品枠の行数」がパラメータとして埋め込まれています。

管理者に次のコマンドで配ってもらいます（自分自身が権限を持っている場合は自分で発行可能）。

```
/vshop egg <対象プレイヤー> <行数 | inf>
```

- `<行数>`: `1` 以上の整数。9 スロット×行数の出品枠を持つショップになります。
    - 1〜6 行のショップは 1 ページで完結します（ページ送りなし）。
    - 7 行以上は 5 行＋ナビ行に分割され、45 スロット単位でページングされます。
- `inf`: 出品枠数が無制限のショップ用エッグ（同じく 45 スロット単位でページング）。

権限 `modernvillagershop.egg` が必要です。

### 1.2 ショップを設置する

1. スポーンエッグを持ち、ブロックの上で右クリックします。
2. 村人が生成され、その場がショップになります。
3. 以下のいずれかに該当すると設置は失敗し、エッグは消費されません。
    - 既存ショップ村人が近すぎる（デフォルトで 0.5 ブロック未満）
    - 所有ショップ数の上限（`shop.maxShopsPerPlayer`）に達している

設置後、村人の頭上には「ショップ名 `[所有者名]`」がデフォルトで表示されます。

### 1.3 ショップを削除する

- ショップ削除は **編集メニュー**（後述）の `ショップを削除` から行います。村人を直接倒しても消えません。
- 在庫が残っているときの挙動は管理者側の設定（`shop.closeWithInventory`）で決まります。既定は **REFUSE**（在庫がある間は削除拒否）です。まず在庫を空にしてから削除します。
- 削除時、権限 `modernvillagershop.edit.delete.refund` があれば、使用時と同等のスポーンエッグが村人の位置にドロップします。

## 2. 編集メニューを開く

自分のショップの管理は「編集メニュー」から行います。以下のいずれかで開けます。

- コマンド `/vshop edit <shopId>` を実行する
- 自分のショップを見て `/vshop edit`（引数省略時、視線先の店舗が対象）を実行する

編集モード中は他プレイヤーからの購入・納品がブロックされます。作業が終わったらメニューを閉じてください。

編集メニューには次のようなアクションが並びます（役割・権限次第で表示されない項目があります）。

- **品目を編集**: 出品枠の追加・変更・削除（チェストUI）
- **プレビュー**: 一般プレイヤー視点の閲覧UIを開いて見え方を確認
- **在庫を編集**: プレイヤーショップの在庫を出し入れ
- **店舗情報 / 店舗設定 / オーナー管理**: 各サブメニューへ
- **取引通知**: あなた個人の通知 ON/OFF 切替
- **一時停止 / 公開を再開**: 状態切替
- **ショップを削除**: 削除フロー

## 3. 出品枠を編集する

編集メニューの `品目を編集` をクリックすると **編集モードのチェストUI** が開きます。閲覧モードとは枠飾りやタイトルで区別されます。

### 3.1 新規に出品枠を作る

1. 空きスロットに、カーソルに持ったアイテムを重ねてクリックします（アイテムは消費されず、参照用にスナップショットされます）。
2. 出品枠設定ダイアログが開きます。以下を設定します。
    - **種別**: `SELL`（販売） / `BUY`（買取） / `BOTH`（双方向）
    - **販売単価** / **買取単価**
    - **1パックあたりの個数**（例: 16 に設定すると、購入者は 16 個単位でしか買えません）
    - **買取受入残**: これ以上は買い取らない上限数量（`-1` で無制限）
    - **取引上限**: 時間あたりの上限（空欄で無制限）
    - **上限スコープ**: `PER_PLAYER`（プレイヤー単位）／`GLOBAL`（全体合算）
    - **上限リセット秒数**: 上限を巻き戻す周期（空欄でリセットなし）
3. `適用` で確定。ダイアログを閉じたり `キャンセル` した場合はスナップショットは破棄され、スロットは空のままです。

### 3.2 既存の出品枠を編集・削除する

- 既存の枠をクリックすると同じダイアログが開き、内容を書き換えられます。
- ダイアログ内の **削除** ボタンで枠を削除できます（確認ダイアログを経由）。

### 3.3 気をつけたい点

- **取引禁止アイテム**（管理者が `items.blacklist` に登録したもの。既定でシュルカーボックス系・バンドル）は出品できません。
- **アイテムの同一性**は厳密判定です。BUY 枠に「エンチャント本」を登録すると、リネームや別レベルのエンチャント本は納品対象外になります。
- 単価は `economy.priceMin`〜`priceMax` の範囲内で、1パックあたりの個数は `economy.amountMax` 以下である必要があります。

## 4. 在庫を管理する（プレイヤーショップのみ）

編集メニューの `在庫を編集` から在庫チェストUIを開きます。管理者ショップには在庫はありません（無限在庫扱い）。

- チェストUIとあなたのインベントリの間で自由にアイテムを移動できます。
- スロット位置はそのまま保存されます（並べ替えて保管庫代わりに使えます）。
- 取引禁止アイテムを入れて閉じた場合、そのアイテムはあなたに返却されます。
- 閉じたタイミングで DB に差分が保存されます。編集モードと同じく、開いている間は他人の購入・納品は保留されます。

BUY 枠に対する納品は在庫に積み上がるので、それを SELL 枠として再販する運用も可能です。

## 5. 名前・職業・公開状態・場所を変える

編集メニュー内の `店舗設定` サブメニューから変更できます。

- **ショップ名変更**: 村人の頭上表示名は `shop.villagerNameFormat`（既定: `<shop_name> [<primary>]`）に沿って自動再生成されます。
- **職業変更**: 村人の見た目（職業ブロック）を変えます。
- **一時停止 / 公開再開**: 一時停止中は購入も納品も拒否されます。長期不在時に使えます。
- **移動**: 権限 `modernvillagershop.edit.move` がある場合、ショップを別の位置に移動できます。

## 6. 共同オーナー（プレイヤーショップのみ）

自分ひとりで運営するだけでなく、複数人でショップを共同運営できます。

### 6.1 役割

| 役割 | 権限の要約 |
| --- | --- |
| **PRIMARY** | 1名固定。ショップ削除、共同オーナー管理、PRIMARY 移譲、および MANAGER の全権限 |
| **MANAGER** | 出品枠の追加・編集・削除、価格・数量変更、在庫補充/引出し、公開・一時停止、名前・職業・移動 |
| **STAFF** | 在庫補充のみ。収益は受け取れず、編集も不可 |

### 6.2 収益分配

- 各オーナーには `share`（持分、小数2桁の百分率）が割り当てられます。合計 100.00 になるよう PRIMARY の持分で自動補正されます。
- SELL 取引の収益は手数料を差し引いた後、`share` に応じて按分入金されます。
- BUY 取引の支払いは **PRIMARY の口座から**行われます。PRIMARY の残高不足時は取引が拒否されます。
- STAFF は `share = 0` として扱われ、収益は入りません。

### 6.3 追加・編集・削除

PRIMARY のみが操作できます（またはサーバー管理者権限 `modernvillagershop.coowner.manage.others` を持つ人）。

```
/vshop coowner <shopId>
```

または編集メニューの `オーナー管理 → 共同オーナー` から開けます。

- **追加**: プレイヤー選択チェストUI（オンライン優先、名前検索・並び替え可）から対象を選び、役割と持分を指定します。未参加のプレイヤーは選択できません（キャッシュ済みプレイヤーのみが対象）。
- **役割・持分変更**: 一覧から対象をクリックして編集します。
- **削除**: 一覧から対象をクリック → `削除`。

### 6.4 PRIMARY 移譲

```
/vshop transfer <shopId> <対象プレイヤー>
```

または編集メニューの `オーナー管理 → PRIMARY 移譲`。確認ダイアログを経て確定します。

- 譲渡元は自動的に MANAGER に降格します。
- 譲渡先が既存の共同オーナーでも、まったく新規のプレイヤーでも可能です。
- 移譲時に村人の表示名が再生成されます。
- 対象プレイヤーがオフラインでも実行できます（次回ログイン時に通知）。

## 7. 通知

自分のショップで取引が発生したときの通知は、編集メニューの `取引通知` ボタンで ON/OFF を切り替えられます。

- **オンライン時**: 取引ごとにチャット通知が届きます（`あなたのショップ ... で ... が売れました` のような形）。
- **オフライン時**: 次回ログイン時に「不在中にあなたのショップで N 件の取引がありました（合計 ...）」のサマリが 1 回だけ届きます。詳細は `/vshop history` で追えます。
- 通知は PRIMARY と MANAGER に届きます。STAFF には届きません。
- 共同オーナーの追加・削除・役割変更・PRIMARY 移譲は、対象プレイヤーに通知が届きます（オフライン時はキュー）。

## 8. 取引履歴を活用する

基本形は `/vshop history` ですが、フィルタで絞り込めます。

```
/vshop history [shopId] [ページ] [--side sell|buy] [--from <日付>] [--to <日付>] [--player <name>] [--page <n>]
```

- **shopId**: 先頭一致で OK（UUID の頭数文字だけでも動きます）。省略すると自分が関与した履歴すべてが対象。
- **--side sell** / **--side buy**: 売買種別で絞る。
- **--from / --to**: `YYYY-MM-DD` または `YYYY-MM-DDTHH:mm[:ss]` で範囲指定。サーバーのタイムゾーンで解釈されます。
- **--player**: 取引相手の名前で絞る（前方一致）。
- **--page**: ページ番号を明示指定（位置引数の 2 番目でも指定可）。

例:

```
/vshop history                         # 自分の全履歴
/vshop history 3a2f                    # 特定ショップ（UUID 頭 4 文字一致）
/vshop history --side sell --from 2026-07-01
/vshop history --player Steve --page 2
```

## 9. 統計を見る

```
/vshop stats <shopId>
```

- 出品枠数 / 稼働中枠数
- SELL・BUY それぞれの累計取引件数と金額
- 累計手数料
- 直近 7 日の取引件数推移
- 人気アイテム Top 5

共同オーナーは全員閲覧できます。

## 10. 動的価格（PriceProvider）の読み方

管理者ショップでは、拡張プラグインが価格を動的に決めている場合があります（プレイヤーショップの単価は静的）。

- チェストUI の各枠 Lore に **理由テキスト**（例: `セール中`）が表示されることがあります。これは価格を上書きした Provider が示すメッセージです。
- 価格は **購入確認ダイアログを開いた時点でその値に凍結**されます（`(価格凍結: <時刻>)` と表示）。
- 確定時に凍結価格から乖離許容率を超えて価格が変動していた場合、取引は自動キャンセルされます。もう一度開き直してください。

## 11. トラブルシューティング（自分のショップ編）

| 症状 | 対処 |
| --- | --- |
| **`/vshop egg` の対象プレイヤーが見つからない** | オンラインかつスペルが正しいことを確認。オフラインプレイヤーには配布できません。 |
| **設置しようとしても「近くにショップがあるため設置できません」** | 既存ショップ村人から少し離れて設置し直してください。 |
| **ショップを削除しようとして「在庫が残っているため閉じられません」** | 先に `在庫を編集` から在庫を空にしてください。挙動は `shop.closeWithInventory` の設定次第です。 |
| **共同オーナーに設定したいプレイヤーがプレイヤー選択UIに出ない** | そのプレイヤーが一度もサーバーにログインしていない可能性があります。プラグインはログイン済みプレイヤー（`player_cache`）のみを候補にします。 |
| **通知が来ない** | 編集メニュー `取引通知` が OFF になっていないか確認。STAFF ロールでは仕様上通知は来ません。 |
| **買取（BUY）で「店主の残高不足」と言われた** | あなたが PRIMARY なら残高を補充してください。共同オーナー体制でも支払いは常に PRIMARY 口座からです。 |
| **村人がテレポート等で動いてしまった** | AI 無効化は入っていますが、ネザーポータル・強制テレポート等は別の要因の可能性があります。座標は DB に保存されているのでチャンクリロード時に元の位置へ戻ることがあります。 |
