# 利用者ガイド（基本機能）

ModernVillagerShop の一般プレイヤー向けガイドです。ここでは「他人が開いているショップを利用する」ための操作をまとめています。自分でショップを持つ／編集する機能は [利用者ガイド（発展機能）](user-advanced.md) を参照してください。

## 1. ショップとは

- ショップは村人（Villager）の姿で世界に配置されています。
- 各ショップは複数の「出品枠（スロット）」を持ち、それぞれに以下のいずれかが設定されています。
    - **販売（SELL）**: 店主が売っているアイテムをあなたが買う
    - **買取（BUY）**: 店主が募集しているアイテムをあなたが納品して代金を受け取る
    - **双方向（BOTH）**: 同じアイテムの販売と買取を1枠で扱う
- ショップは「公開中」か「一時停止中」の状態を持ちます。一時停止中は購入も納品もできません。

## 2. ショップを開く

### 村人を右クリックして開く

- ショップ化された村人を近くで右クリックすると、その場でチェストUIが開きます。
- 開けない場合、以下のいずれかが原因です。
    - 距離が遠い（デフォルトは `shop.openDistance` = 6ブロック以内）
    - ショップが一時停止中
    - サーバー管理者があなたに `modernvillagershop.use` 権限を付与していない

### コマンドで開く

`/vshop open <shopId>` で ID を指定して開けます。`shopId` は UUID の先頭数文字でも一致します。距離制限を無視して開けるかどうかは権限次第です（`modernvillagershop.open.nearby` が既定、`modernvillagershop.open.any` を持つと距離無視）。

## 3. ショップを探す

### 一覧を見る

```
/vshop list [ページ番号]
```

サーバー上のショップを一覧表示します。ページ番号を省略すると 1 ページ目が表示されます。

### アイテム名で検索する

```
/vshop search <アイテム名またはID> [ページ番号]
```

- 例: `/vshop search diamond`、`/vshop search minecraft:iron_ingot`
- 販売中・買取中のアイテム名で部分一致検索します。ヒットしたショップのIDと単価、種別（SELL / BUY）が表示されます。
- 見つけたショップは `/vshop open <shopId>` で開くか、実際にその位置まで移動して村人を右クリックします。

## 4. 購入する（SELL 枠）

1. ショップを開くと、チェストUIに出品枠のサンプルアイテムが並びます。
2. 目当ての枠をクリックします（BOTH 枠の場合は「購入する / 納品する」の選択画面が出るので `購入する` を選びます）。
3. **数量入力ダイアログ**が開きます。1パックの個数がショップ側で決まっているので、購入したい **パック数** を入力します。
    - 例: 1パック = 16個の販売枠で 3 パック買うと合計 48 個
4. 続いて **購入確認ダイアログ**で、アイテム・数量・単価・合計金額・手数料が表示されます。よければ `購入する`。
5. 成功すると代金があなたの残高から即時引き落とされ、アイテムがインベントリに入ります。

### 購入できないとき

- **在庫が不足しています**: 店の在庫がその数量に足りません。
- **所持金が足りません**: あなたの残高が代金に足りません。
- **取引上限に達しています**: この枠には時間あたりの購入上限が設定されています。しばらく待つと解除される場合があります。
- **価格が変動したためキャンセルしました**: 管理者ショップで動的価格が使われている場合、確認画面を開いたあとに価格が変わり過ぎると自動キャンセルされます。もう一度開き直してください。
- **このスロットは現在取引できません**: 店主が編集中、またはショップが一時停止中です。

### インベントリが埋まっていた場合

購入したアイテムはあなたのインベントリに直接入ります。空きが足りないと購入が失敗する可能性があるため、事前に空きを確保しておいてください。

## 5. 納品する（BUY 枠）

BUY 枠は「あなたが持っているアイテムを店主が買い取ってくれる」枠です。

1. ショップを開き、BUY 枠（買取単価がロアに表示されている枠）をクリックします。
2. BOTH 枠の場合は `納品する` を選びます。
3. 数量入力ダイアログでパック数を入力します。
4. 納品確認ダイアログで受取額と手数料を確認し、`納品する` を押します。
5. 成功するとあなたのインベントリからアイテムが減り、代金が即時入金されます。

### 納品できないとき

- **納品するアイテムが手持ちに足りません**: あなたのインベントリに対象アイテムが必要数ありません。
- **受入枠が満杯です**: 店主が「これ以上は買い取らない」上限に達しています。
- **店主の残高が不足しているため取引できません**: プレイヤーショップの場合、店主の所持金が支払いに足りていません。店主にお知らせしましょう（管理者ショップではこのメッセージは出ません）。
- **取引上限に達しています**: 時間あたりの納品上限に達しています。

## 6. 取引履歴を見る

自分の取引履歴は次で確認できます。

```
/vshop history
/vshop history 2         # 2 ページ目
```

- 直近の売買・納品が新しい順に一覧されます。
- より詳細なフィルタ（期間・相手・売買種別など）は [利用者ガイド（発展機能）](user-advanced.md) を参照してください。

## 7. アイテムの同一性について

「同じアイテムかどうか」は名前・エンチャント・カスタムデータまで含めて厳密に判定されます（Bukkit の `ItemStack#isSimilar`）。以下は別物として扱われる例です。

- エンチャントの有無や種類・レベルが違うツール
- リネーム済み／未リネームのアイテム
- カスタムモデルデータが違う装飾アイテム

そのため「見た目は同じでも売れない・買い取ってもらえない」場合があります。ショップの表示アイテムをまるごと持ち込むのが確実です。

## 8. よくある質問

- **買った村人（ショップ）を叩いても消えません**: ショップ用の村人は無敵で、削除は店主・管理者だけができます。仕様通りです。
- **通知が来ない**: 通知の受け取りはショップの店主・共同オーナー向けの機能で、購入者側には送られません。自分のショップの通知については発展編を参照してください。
- **Bedrock 版で装飾（色）が出ない**: Bedrock ではダイアログの装飾がプレーンテキストにフォールバックする仕様です。表示は簡素になりますが操作は同じです。
